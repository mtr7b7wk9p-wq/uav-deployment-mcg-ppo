# Resource Cognition Design

## Goal

在不破坏现有用户覆盖实验的前提下，为灾后无人机完全分布式资源调度课题建立独立的可信状态认知与协同感知模式。

## Research Alignment

课题第一章的技术链路为：灾后信息不完备 -> 无稳定中心和预设簇头 -> 局部可信状态认知 -> 按需协同感知 -> 频谱、能量、需求协同调度。

现有 `DisasterDeploymentEnv` 保留为覆盖原型。新增资源认知链路不再把覆盖用户作为感知状态本身，而是使用显式任务单元表示待更新的资源状态。第一阶段先采用区域任务加单频段的最小模型，后续再扩展到多频段和调度执行。

## Architecture

- `envs/task_model.py`: 定义任务状态、可信度、不确定性和信息年龄更新，不依赖 PPO。
- `envs/observation_model.py`: 把任务和邻居状态转换为局部、带时效的信息，不暴露全局真实状态。
- `envs/resource_cognition_env.py`: 负责任务采样、移动、感知动作、奖励和终止条件；复用已有几何工具，但不修改覆盖环境语义。
- `configs/scenario_config.py`: 只保存资源认知参数和维度配置。
- `envs/metrics.py`: 保留旧覆盖指标，并增加认知质量、AoI、重复感知和需求优先级指标。

## First-Phase Semantics

资源认知动作使用固定离散接口：`stay`、四个移动动作、`sense_task_slot_1...K`。移动不自动产生认知收益，只有显式感知动作才能更新任务状态。不可见任务不能被直接选择；多个无人机选择同一任务时才计算重复感知。

每个任务至少包含：空间位置、频段标识、真实状态、估计状态、不确定性、AoI、需求优先级和可信度。第一阶段的观测只包含局部可见任务和有限邻居摘要；全局状态只用于训练器或调试，不进入单机策略输入。

## Compatibility

- `ppo_main`、`mcg_ppo` 和现有覆盖基线的默认配置不变。
- 新模式使用独立方法名，不覆盖旧的 `mcg_ppo` 输出目录。
- 旧覆盖指标字段继续保留；新指标使用独立字段名。

## Validation

- 旧覆盖模式的 observation/action 维度和最小训练流程保持可运行。
- 资源认知模式中，移动不会更新任务不确定性，显式感知才会更新。
- 不同任务选择不会产生重复感知，重复选择会产生重复率。
- 相同种子产生相同任务初始状态。
- 资源认知模式可完成一次 CPU 最小训练并输出认知指标。
